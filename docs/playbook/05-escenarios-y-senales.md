# 05 — Escenarios y señales (Colombia)

## Tres escenarios (obligatorio)

Para el **acto macro** (y, si aplica, para casos prioritarios):

1. **Base** — trayectoria más plausible con la información actual  
2. **Optimista** — mejora relativa bajo condiciones explícitas  
3. **Pesimista** — deterioro bajo condiciones explícitas  

Cada uno con:

- Narrativa (horizontes 90/365)
- Condiciones
- **Señales de confirmación**
- **Señales de invalidación**

Plantilla metodológica: [`docs/metodologia/02-plantilla-escenarios.md`](../metodologia/02-plantilla-escenarios.md).

## Relación con casos

- Casos de alta credibilidad + baja factibilidad empujan el escenario **pesimista** de decepción o choque institucional si no hay capacidad.
- Casos identidad marcan la narrativa política aunque la frecuencia de otros temas sea mayor.
- Mencionar en el informe los `caso_id` relevantes (no inventar %).

## Actualizar señales

Cuando entre un evento/mención nueva:

1. ¿Confirma o invalida un escenario?
2. Escribir en `senales` con `caso_id` opcional, fecha, URL.
3. Recalcular credibilidad del caso si aplica (WF-D).
4. No archivar mención única importante.

## Límites

Declarar qué no se puede afirmar (salud del presidente, guerra externa, choques de commodities, etc.).
